المتطلبات المسبقة

قبل البدء في استضافة موقع الويب الخاص بك على GitHub، ستحتاج إلى:

  • حساب GitHub مطلوب
  • مستودع GitHub مع ملفات موقع الويب الخاص بك مطلوب
  • ملفات HTML و CSS الأساسية مطلوب

ملاحظة مهمة

تعمل خدمة GitHub Pages بشكل أفضل مع مواقع الويب الثابتة التي تستخدم HTML و CSS و JavaScript. إنها ليست مناسبة لتطبيقات الواجهة الخلفية (Backend).

الخطوة 1: الوصول إلى إعدادات GitHub

  1. افتح مستودع GitHub الذي يحتوي على ملفات موقع الويب الخاص بك.
  2. انقر فوق علامة التبويب Settings (الإعدادات) الموجودة في أعلى صفحة المستودع.

تنبيه

يجب أن تكون مالك المستودع أو أن يكون لديك أذونات المسؤول للوصول إلى إعدادات المستودع.

الخطوة 2: تكوين GitHub Pages

  1. في صفحة الإعدادات، قم بالتمرير لأسفل وانقر على Pages في قسم Code and automation.
  2. ضمن Source، اختر Deploy from a branch.
  3. اختر الفرع (Branch) الذي يحتوي على التعليمات البرمجية التي تريد نشرها (عادةً main أو master).
  4. حدد الدليل (Directory) حيث توجد ملفاتك. اختر / (root) إذا كانت ملفاتك في الدليل الجذر للمستودع.
  5. انقر فوق زر Save (حفظ) للحفظ والبدء في عملية النشر.

نصيحة احترافية

في معظم المشاريع الصغيرة، يتم وضع ملفات موقع الويب في الدليل الجذر (/). ومع ذلك، يمكنك أيضًا اختيار الدليل /docs إذا كنت تفضل الاحتفاظ بملفات موقعك منفصلة.

الخطوة 3: مراقبة عملية النشر

  1. ارجع إلى علامة التبويب Code في مستودع GitHub الخاص بك.
  2. ستلاحظ دائرة صفراء بجوار آخر التزام (commit). انقر عليها لمعرفة حالة النشر.
  3. انقر فوق Details لعرض حالة عملية النشر بالتفصيل.
  4. انتظر حتى تكتمل جميع الخطوات بنجاح (سيستغرق ذلك عادةً دقيقة أو دقيقتين).

عملية النشر

تقوم GitHub Pages تلقائيًا بما يلي:

  • جلب أحدث محتوى من الفرع المحدد
  • بناء موقع الويب
  • نشر الموقع على خوادم GitHub Pages
  • إنشاء رابط عام لموقع الويب الخاص بك
100%

الخطوة 4: الوصول إلى الموقع المنشور

  1. بعد اكتمال عملية النشر، عد إلى قسم Settings > Pages.
  2. ستجد رابط موقع الويب المنشور الخاص بك في أعلى الصفحة.
  3. يمكنك النقر على هذا الرابط أو اختيار Visit site للوصول إلى موقع الويب المنشور الخاص بك.

مشاركة موقعك

إلغاء نشر الموقع

إذا كنت ترغب في إلغاء نشر موقع الويب الخاص بك في أي وقت، يمكنك اتباع هذه الخطوات البسيطة:

  1. انتقل إلى صفحة Settings > Pages في مستودع GitHub الخاص بك.
  2. انقر على النقاط الثلاث (⋮) في الزاوية اليمنى العليا.
  3. اختر خيار Unpublish site (إلغاء نشر الموقع).

تنبيه

سيؤدي إلغاء نشر موقعك إلى جعله غير متاح على الإنترنت، ولكنه لن يؤثر على ملفات المصدر في مستودع GitHub الخاص بك.

الخلاصة

تعد GitHub Pages طريقة رائعة ومجانية لاستضافة مواقع الويب الأمامية (Front-end) الخاصة بك. تتيح لك عرض مشاريعك، وإنشاء مواقع شخصية، وحتى نشر المدونات والوثائق، كل ذلك دون تكلفة الاستضافة التقليدية.

باتباع هذا الدليل البسيط، يمكنك نشر موقع ويب خاص بك عبر الإنترنت في دقيقتين فقط.

العودة إلى الأعلى

الأسئلة الشائعة

نعم، يمكنك استخدام اسم نطاق مخصص مع GitHub Pages. بعد نشر موقعك، يمكنك الانتقال إلى إعدادات GitHub Pages وإضافة اسم النطاق المخصص الخاص بك في قسم "Custom domain".

GitHub Pages مخصص بشكل أساسي لمواقع الويب الثابتة (HTML، CSS، JavaScript). لا يمكنه استضافة تطبيقات الواجهة الخلفية (Backend) التي تتطلب خوادم مثل PHP، Node.js، أو Python. ومع ذلك، يمكنك استخدام إطار عمل JavaScript مثل React أو Vue لإنشاء تطبيقات واجهة أمامية (Frontend) ديناميكية.

تشمل حدود GitHub Pages حجم مستودع يبلغ 1 جيجابايت، وحد بناء شهري يبلغ 100 جيجابايت، و10 عمليات بناء في الساعة. هذه الحدود كافية للمواقع الشخصية ومواقع المشاريع النموذجية.
